# Quiz Clash

Kahoot風のリアルタイムクイズ対戦アプリです。1人がホストとなってクイズを作成・出題し、複数人が自分のスマホやPCから参加してリアルタイムに回答・採点されます。

## 重要: GitHub Pagesでは動きません

このアプリは Node.js + Socket.IO のサーバー（`server.js`）がルーム管理・採点を行う仕組みです。**GitHub Pagesは静的ファイル（HTML/CSS/JS）しか配信できず、Node.jsサーバーを実行できないため、GitHub Pagesにアップロードしただけでは動作しません。** インターネット上に公開するには、後述の「クラウドへの公開（Render）」の手順でNode.js対応のホスティングにデプロイしてください。

## ローカルで試す

```bash
cd kahoot-clone
npm install
npm start
```

`http://localhost:3000` がホスト用、別デバイスから参加する場合はホストと**同じWi-Fi**に接続し、ホストPCのローカルIPアドレス（`ipconfig`/`hostname -I`などで確認）を使って `http://<ホストのIP>:3000/play.html` にアクセスします。

## クラウドへの公開（Render）

同じWi-Fiでなくても、インターネット経由でどこからでも参加できるようにするには、Node.js対応の無料ホスティング **[Render](https://render.com)** にデプロイするのがおすすめです（GitHub Pagesとは別物です）。

1. このフォルダの内容をGitHubリポジトリにpushする（GitHub Pages用に使っていたリポジトリでも、新しいリポジトリでもOK。Pages機能自体は使いません）
2. [render.com](https://render.com) にアクセスし、GitHubアカウントでサインアップ・ログイン
3. ダッシュボードで「New +」→「Web Service」を選択し、対象のGitHubリポジトリを連携
4. 設定項目を入力（リポジトリに含まれている `render.yaml` を検出した場合は自動入力されます）
   - Build Command: `npm install`
   - Start Command: `npm start`
   - Instance Type: **Free**
5. 「Create Web Service」でデプロイ開始。数分後に `https://quiz-clash-xxxx.onrender.com` のようなURLが発行されます
6. ホストはこのURLの `/host.html`、参加者は `/play.html` にアクセス（参加者は同じWi-Fi不要、インターネットに繋がっていればOK）

### Renderの無料プランに関する注意

- 15分間アクセスがないとサービスがスリープし、次のアクセス時に再起動のため約1分かかります（クイズ開催の直前に一度URLを開いて起こしておくと安心です）
- 無料枠は1ワークスペースあたり月750時間（常時起動でなければ十分な枠です）
- ゲーム中にプレイヤーの接続が続いていればスリープしないため、開催中に途切れる心配は基本的にありません

## 使い方

### 1. ホスト（出題する人）

1. トップページ（ローカルなら`http://localhost:3000`、Render公開後ならRenderが発行したURL）を開く
2. 「クイズを作成してホストする」を選択
3. クイズのタイトル・問題・選択肢・正解・制限時間を入力（「サンプルを読み込む」で例を試せます）
4. 「ゲームを作成する」を押すと6桁の**参加コード**が表示されます
5. 参加者が集まったら「クイズを開始する」を押して進行します

### 2. 参加者（回答する人）

1. ホストが案内したURLの `/play.html` を開く（例: `https://quiz-clash-xxxx.onrender.com/play.html`）
2. ホスト画面に表示された参加コードとニックネームを入力して参加
3. ホスト画面に出題された問題を見ながら、自分の画面の図形（▲◆●■）をタップして回答

## ゲームの仕組み

- 回答が早いほど高得点（正解時 500〜1000点、不正解・未回答は0点）
- 各問題ごとに結果発表→リーダーボード表示→次の問題、という流れで進行
- 参加者の画面には問題文ではなく図形のみが表示されます（Kahootと同様、ホスト側の画面を見て答える方式）
- 全問終了後、表彰台（上位3名）と最終ランキングを表示

## 既知の制限事項

- データはメモリ上のみで保持されるため、**サーバーを再起動するとゲームは消えます**（Renderのスリープ復帰時も同様にリセットされます）
- ホストがブラウザをリロード・切断すると、そのゲームは終了します
- 同時に複数のクイズ（ルーム）を並行して開催することは可能です（参加コードで区別されます）

## ファイル構成

```
kahoot-clone/
├─ server.js              # Express + Socket.IO サーバー（ルーム管理・採点ロジック）
├─ package.json
├─ render.yaml            # Renderへのデプロイ設定（任意）
├─ public/
│  ├─ index.html          # トップページ（ホスト/参加の選択）
│  ├─ host.html           # ホスト用画面（作成・進行）
│  ├─ play.html           # 参加者用画面（参加・回答）
│  ├─ css/style.css
│  └─ js/host.js, play.js
└─ quizzes/sample-quiz.json  # サンプルクイズ
```
